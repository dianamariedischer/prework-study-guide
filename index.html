HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The footer element is another section of visible content, but at the bottom of the webpage.
  • The meta element can be used for:
    • SEO
    • Setting a character encoding system (UTF-8 is common)
    • Viewport settings
    • And like other stuff haha
  • HTML creates the structure of the webpage.
  • A <link> element connects to an external file, rel defines the relationship, and href
    points to the file location.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Structure of a CSS rule: selector/element to which the rules apply {property: value}
    • "*" selector applies rule to all visible elements.
  • If you assign block as the display value for an image, it will start the image on a new line.
  • You can create a class selector by preceding a word with a period. Any HTML element assigned to this class gains the properties from the CSS.
  • The box-shadow CSS property adds shadow effects around an element's frame. You can set multiple effects separated by commas. A box shadow is described by X and Y offsets relative to the element, blur and spread radius, and color. In our example, the X offset is 5px and defines the width or extent of the shadow on the X axis (or to the right of the box); the Y offset is 10px and defines the extent on the Y axis (or just below the box).

Git

Commands:

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
    -b creates a new branch
  • git checkout main: takes you back to the main branch
  • git pull origin branch-name: pulls named branch into current branch?
    Good practice to do this before pushing a branch for a pull request into main.
  • git push origin branch-name: I think this pushes a branch up for review in github?
  • git add file-name: adds file to staging - Use -A for all.
  • git commit -m "decription of adjustments": commits staged files with a message
    When you add -m it allows you to attribute a message in quotes to the commit.

JavaScript

  • var creates a variable that you need to define.
  • console.log() is a good way to test code and debug.
  • Creating an if statement- if (condition) {code to be executed if true}
  • === is the strict equality operator, returns a boolean (true or false).
  • Arrays hold related data.
  • Control flow is the order in which a computer executes code in a script.
